# Express + TypeScript Modular Boilerplate

An industry-level, highly scalable Express boilerplate built with TypeScript. Inspired by modern frameworks like **NestJS** and **AdonisJS**, this boilerplate is organized into a modular folder structure that enforces single-responsibility and separation of concerns.

## 🚀 Features

- **TypeScript** & **Express** out of the box.
- **Swagger Documentation** pre-configured.
- **Winston Logger** for professional-level observability.
- **Modular Directory Structure** (Controllers, Services, Routes, Middlewares).
- **Interactive Database CLI**: Easily spin up MongoDB, PostgreSQL, or MySQL and generate ORM templates effortlessly.

## 📂 Folder Structure

```text
src/
├── app.ts                 # Express application setup, middlewares, and route injection
├── server.ts              # Entry point to start the server
├── config/                # Environment variables and DB configurations
├── controllers/           # Route logic handlers (e.g. HealthController)
├── routes/                # Express router definitions
├── services/              # Business logic layer
├── middlewares/           # Custom Express middlewares (Auth, Error handling)
├── utils/                 # Utility functions (logger, formatters)
├── interfaces/            # Global TypeScript definitions and Types
├── entities/ (or models)  # Database ORM/ODM models (Auto-generated by CLI setup)
└── swagger.ts             # Swagger automated API docs setup
```

## 🛠 Getting Started

### 1. Install Dependencies
*(assuming you ran the creation process successfully)*

### 2. Start the Server

```bash
npm run dev
```

The server will start at `http://localhost:3000`.
Visit `http://localhost:3000/api-docs` to view the swagger API documentation.

### 3. Setup Your Database & Components (Interactive CLI)

We provide an AdonisJS/NestJS inspired interactive CLI.

```bash
npm run ace
# or
npm run cli
```

The CLI wizard will allow you to pick:
- **Setup Database**: Choose from Mongoose, Prisma, TypeORM, Sequelize, MikroORM, Knex.js, or Lucid.
- **Generate Controller**: Automatically scaffold a class-based controller in `src/controllers/`.

Once you set up a DB, just uncomment `// await connectDB();` in `src/server.ts` to connect to it on boot!

## 🧑‍💻 Development Workflow

1. Create a `*.route.ts` file in `src/routes`.
2. Map it to `src/app.ts`.
3. Create the Handler/Controller in `src/controllers`.
4. Keep complex logic within `src/services`.
5. Check your types within `src/interfaces`.
